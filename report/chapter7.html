Chapter 7: System Testing

Verification and Validation (V&V) confirm that the software system matches its SRS specifications. Testing was executed at three levels: unit testing of models, integration testing of API endpoints, and validation testing of client workflows.

7.1 Unit Testing

Unit tests verify the correctness of isolated, small pieces of logic — primarily database constraints, model methods, and helper utilities.

Sample Django Model Unit Test Case

backend/users/tests/test_models.py
Python
from django.test import TestCase
from users.models import User

class UserModelTest(TestCase):
    def setUp(self):
        self.user = User.objects.create_user(
            email="vendor@test.com",
            username="testvendor",
            password="secret123",
            role="vendor"
        )

    def test_role_assignment(self):
        self.assertEqual(self.user.role, "vendor")
        self.assertTrue(self.user.is_active)

    def test_password_hashing(self):
        self.assertTrue(self.user.check_password("secret123"))
        self.assertNotEqual(self.user.password, "secret123") # check hashing

7.2 Integration Testing

Integration tests verify the communication channels between subsystems — such as verifying endpoints retrieve correctly from tables and respect user access tokens.

Sample DRF API client integration test

backend/products/tests/test_apis.py
Python
from rest_framework.test import APITestCase
from rest_framework import status

class ProductApiTest(APITestCase):
    def test_public_product_listing(self):
        response = self.client.get("/api/products/")
        self.assertEqual(response.status_code, status.HTTP_200_OK)

    def test_customer_blocked_write(self):
        # Customer role is authenticated but RBAC blocks write = 403
        self.client.force_authenticate(user=customer_user)
        response = self.client.post("/api/products/", {"name": "Pen", "price": 10})
        self.assertEqual(response.status_code, status.HTTP_403_FORBIDDEN)

7.4 Debugging Log

Below are five notable developer bugs resolved during final system integration.

Bug / Symptom Root Cause Resolution Applied
CORS Policy Blocking API Requests Browser blocked requests from localhost:5173 to localhost:8000 because of missing headers. Installed django-cors-headers, added CorsMiddleware first, and set CORS_ALLOWED_ORIGINS.
Protected Endpoints return 401 JWT tokens were stored in local storage but never attached to outgoing Axios calls. Added a request interceptor in src/services/api.js to inject access token into Authorization header.
RelatedObjectDoesNotExist on Product Save A vendor profile record (holding shop name and approval status) was not created when a User registered with role='vendor'. Implemented a Django post_save signal in vendors/models.py to automatically initialize profiles.
IntegrityError: price NOT NULL in OrderItem Order serializer failed to write price at purchase time into OrderItem rows. Modified create() in OrderSerializer to extract the product price snapshot and write it to OrderItem rows.
Cart Totals Showing NaN in Browser Django decimal prices are serialized as strings in JSON. React JS was treating them as strings, causing arithmetic failures. Wrapped all cart calculations with parseFloat() in CartContext.jsx and product card elements.
